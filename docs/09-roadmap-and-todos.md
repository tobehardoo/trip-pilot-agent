# 产品路线图与 TODO

本路线图记录 TripPilot 的产品能力、工程演进方向和公开交付标准。

## 1. 当前可用版本

V1 已完成一条可部署的端到端链路：

- Spring Security 注册、登录、JWT Access Token 与 HttpOnly Refresh Cookie。
- 旅行、结构化约束、规划任务和不可变行程版本持久化。
- Transactional Outbox、RabbitMQ 命令/事件与幂等消费。
- Python Worker、Demo/高德 Provider、Redis 缓存和版本化消息契约。
- 候选 POI 过滤、去重、偏好排序和 OR-Tools 硬约束求解。
- 结构化规划失败、最小放宽建议和跨服务协作取消。
- 广州知识导入、pgvector 检索、来源证据与新鲜度标记。
- Vue 地图、时间轴、SSE 进度、断线补发和降级展示。
- Docker Compose、Prometheus、CI、备份与恢复手册。

## 2. 近期演进

### 规划体验

- 支持活动删除、移动、锁定和影响范围分析。
- 支持局部重规划，并保持未受影响日期和锁定活动不变。
- 增加行程版本比较、回滚和更清晰的冲突修改入口。
- 扩展餐饮时间窗、休息节奏和更多同行类型约束。

### 真实数据

- 接入经过授权的天气、票价、营业时间和预约 Provider。
- 扩展高德公共交通路线与受控路线矩阵。
- 为更多城市建立可审核、可追溯的知识资料。
- 对动态事实提供失效检测和出发前核验提醒。

### 智能规划

- 评测真实语义 Embedding 和可选 Rerank 模型。
- 在固定评测门禁通过后接入受控的模型生成/修复步骤。
- 增加规划 Trace、工具调用耗时、缓存命中率和失败分类。
- 使用固定数据集持续评估可行率、检索质量和约束违反率。

## 3. 中长期能力

### 交通与旅行

- 自驾路线、停车点、停车费用、拥堵和限行。
- 多城市联程。
- 合法的实时酒店、车票和航班 Provider。
- 多套完整行程候选及可解释对比。

### 协作与输出

- 只读分享链接和细粒度访问权限。
- 多人协作与变更记录。
- PDF 或其他格式导出。
- 面向移动端的离线行程查看。

### 平台工程

- 完善 readiness、告警和分布式 Trace。
- 增加死信任务管理与人工重放入口。
- 按实际吞吐评估 CDC Outbox 或托管消息服务。
- 按数据规模评估混合检索和向量索引。
- 只有部署规模证明必要时才引入 Kubernetes。

## 4. 决策原则

- 硬约束校验、幂等和数据所有权不能由模型输出替代。
- 新 Provider 必须具备超时、重试、限流、缓存和错误分类。
- 模型、权重、阈值和索引参数必须通过固定评测集决定。
- 外部动态数据必须标明来源、采集时间和估算属性。
- 新功能应先完成一个可测试的端到端切片，再扩大覆盖范围。
- 不为扩大技术栈而增加没有明确产品价值的组件。

## 5. 公开交付标准

- README 能说明产品价值、技术架构和快速部署方式。
- `.env.example` 只包含变量名和无效示例，不提交真实凭据。
- Demo 模式可以在没有第三方 Key 的情况下复现核心规划流程。
- 关键契约、架构决策、测试门禁和已知边界均有文档。
- 发布前执行自动化测试、Compose 配置校验和敏感信息扫描。
