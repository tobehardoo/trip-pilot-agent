# 项目定位与范围

## 1. 项目目标

构建一个面向国内自由行用户的约束驱动型智能旅行规划平台。用户输入目的地、日期、预算、同行人员、兴趣偏好和固定安排后，系统结合真实 POI、天气、路线、城市知识和约束优化生成可执行的多日行程。

项目需要同时覆盖三类工程能力：

- 业务后端：领域建模、数据库、缓存、异步消息、幂等、并发控制、安全、测试和部署。
- 智能规划：结构化输出、工具调用、RAG、模型适配、成本控制、评测和可观测性。
- Agent 工作流：状态机、任务分解、checkpoint、人工确认、故障恢复和局部重规划。

## 2. 核心产品承诺

系统不只是“根据一句话生成旅游攻略”，而是：

1. 把自然语言需求转换成结构化硬约束、软约束和上下文约束。
2. 使用真实存在的地点和路线，降低地点幻觉。
3. 使用确定性规则和 OR-Tools 校验时间、路线、预算与固定安排。
4. 无法满足硬约束时解释冲突并等待用户选择，不静默放宽。
5. 用户局部修改后，只重规划受影响范围，并保留历史版本。
6. 记录每次 Agent 执行的步骤、外部调用、耗时、Token、成本与失败原因。

## 3. 目标用户和典型场景

- 核心用户：18 至 35 岁国内自由行用户。
- 同行类型：独自、情侣、朋友。
- 常见时长：2 至 5 天；系统上限暂定 7 天。
- 第一版以单个目的地城市为主。
- 用户已经知道到达和返程地点、时间，系统不负责真实购票。
- 用户可以指定酒店；未指定时系统推荐住宿区域，不承诺实时房价和房态。

## 4. 城市覆盖

- 全国城市：依赖高德 POI 和路线数据提供基础规划能力。
- 增强城市：广州、杭州、西安。
- 每个增强城市先维护约 30 个重点 POI，功能稳定后扩充到 50 个。
- 增强信息包括商圈、饮食、文化、季节、建议游玩时长、适合人群、预约提示和旅行注意事项。

三个增强城市用于覆盖不同规划特征：

- 广州：美食密集、老城区、商圈和城市公共交通。
- 杭州：自然景观、预约景点、步行与公共交通组合。
- 西安：历史文化、热门景点、排队与行程强度。

## 5. V1 必做范围

- 用户注册登录和长期旅行偏好。
- 创建旅行并填写日期、预算、同行人员、到达与返程信息。
- 结构化追问缺失信息。
- 异步规划任务、SSE 进度展示和任务取消。
- 高德 POI、地理编码、步行与公共交通路线。
- 城市知识 RAG 和推荐来源展示。
- 候选 POI 过滤、偏好排序和 OR-Tools 约束优化。
- 地图与时间轴联动。
- 删除、移动、锁定活动和局部重规划。
- 不可变行程版本、版本比较和回滚。
- 只读分享链接。
- Agent 运维页、成本统计、失败重试和死信任务处理。
- 基础自动化测试、Agent 评测和本地演示模式。

## 6. 明确不做

- 不爬取 12306、携程等平台的受限实时数据。
- 不提供真实酒店、车票、机票预订、支付、退款。
- 不承诺实时酒店价格、房态或车票余量。
- 不自动预约景点；只提供提醒，用户录入的预约时间作为硬约束。
- V1 不支持自驾、停车、限行和租车。
- V1 不支持多城市联程。
- V1 不支持多人实时协作编辑。
- 不使用多个互相自由对话的“专家 Agent”制造表面复杂度。
- 不让 LLM 直接决定最终数据是否有效。

## 7. 关键产品规则

- 默认交通方式为步行和公共交通。
- 出租车只作为无法满足时间约束时的降级建议，并标明费用为估算。
- 用餐使用固定时间窗，在附近推荐最多 3 家餐厅，不默认锁定某一家。
- 每次生成一个主行程，在冲突点或部分时段提供备选项。
- 硬约束冲突时必须向用户解释并等待选择。
- 实时天气只在数据服务可提供的预报窗口内使用；远期旅行使用季节性知识并标明不确定性。
- 所有估算数据必须展示来源类型，不能伪装成官方实时数据。

## 8. 核心演示路径

1. 用户创建“广州 4 日自由行”。
2. 输入预算、同行类型、兴趣、必去地点和到返时间。
3. Agent 发现缺失信息并返回结构化追问。
4. 用户补充后，页面通过 SSE 展示 POI 搜索、路线计算和优化进度。
5. 地图与时间轴展示结构化行程、通勤、预算和推荐理由。
6. 用户删除第二天一个活动并锁定另一个活动。
7. Agent 只重规划第二天，其他日期保持不变。
8. 用户查看版本差异并回滚。
9. 运维页查看完整 Agent Trace、Token、耗时、缓存和工具错误。

## 9. 成功标准

- 演示中的 POI 均真实存在。
- 固定评测集中的硬约束不得被静默违反。
- 锁定活动不得被局部重规划意外移动。
- 重复消息不得导致重复行程版本。
- 外部服务异常时有明确重试、降级或失败状态。
- 一条命令或明确步骤可以启动本地演示环境。
- README、架构图、在线演示或演示视频足以让使用者和维护者快速理解项目价值。
