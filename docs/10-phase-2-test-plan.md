# Phase 2 测试计划：认证与旅行基础业务

## 范围

- PostgreSQL/Flyway/MyBatis 持久化。
- 注册、登录、JWT Access Token。
- Refresh Token 哈希存储、轮换和重放拒绝。
- 当前用户查询。
- 旅行创建、查询、修改和结构化约束。
- 用户只能访问自己的旅行。

## 测试策略

- 使用 Testcontainers PostgreSQL，不使用 H2 替代真实 SQL 行为。
- HTTP 集成测试覆盖所有本阶段公开 API。
- 每个测试独立清理业务数据，不依赖执行顺序。
- 不 Mock Mapper、数据库、密码编码或 Token 逻辑。
- JaCoCo 在 Maven `verify` 阶段要求至少 80% 行覆盖率。

## 必须覆盖

- 注册成功与邮箱重复。
- 登录成功与错误密码。
- Access Token 访问受保护资源。
- Refresh Token 轮换、过期、撤销和重放。
- 未认证访问返回 401。
- 创建旅行及结构化约束持久化。
- 日期、预算和目的地参数校验。
- 查询旅行列表和详情。
- 跨用户访问返回 404，避免泄露资源存在性。
- 乐观锁版本冲突返回 409。
- 更新约束必须显式携带版本，且固定安排元素、节奏和预算精度在进入数据库前完成校验。
- 旅行详情深链可在会话恢复后直接加载，返回列表时补齐列表数据。
- 约束编辑完整替换时保留固定安排，409 后保留表单并可重新加载最新版本。

## 质量门槛

- 全部测试确定性通过，无重试掩盖失败。
- Maven `verify`、JaCoCo、GitHub CI 全部通过。
- 无真实 Key、Token 或密码进入仓库和日志。
- 集成测试在 10 分钟内完成。
