## 验收范围

### 攻略证据快照

- 旅行所有者可以启用或停用某个攻略来源，其他用户不可操作。
- 创建规划任务时，只快照已启用且尚未过期的攻略事实。
- 快照包含来源、事实身份、观察时间、过期时间和置信度，独立持久化到规划任务审计数据；Python 不读取 Java 业务表。
- 攻略事实只能影响候选排序；不能替代固定安排、预算等硬约束。
- 只有明确正向的事实可以加分；负面提及不反向推广地点，软加权组合不可行时会尝试备选组合。
- 先求得不依赖社区事实的可行基线，再在同一硬约束下尝试情报优化；情报优化失败必须返回基线。
- 影响规划的攻略事实作为可点击来源进入最终知识引用。

### 旅行上下文 V2

- 支持到达与返程地点/时间、住宿锚点、必去地点、排除地点、每日用餐窗口和行动能力。
- 旧版未提供 V2 字段的请求使用安全默认值，已有旅行可平滑迁移。
- 到返时间必须带时区并位于旅行日期范围内；返程不得早于到达。
- 用餐窗口必须有序、互不重叠、不得重复同一餐别；半填写字段不允许被前端静默丢弃。
- 规划器把必去地点作为硬优先，过滤排除地点。
- 首日到达、住宿、末日返程的实际接驳耗时与用餐窗口进入排程；减少步行时尝试更近组合，无台阶偏好使用车行接驳并明确场地无障碍仍需确认。
- 当前单次旅行限制为 1 至 7 天；跨天地点组合必须回溯验证，候选、组合尝试和路线调用均有硬上限与任务内缓存。
- Provider 降级不得绕过硬约束：Demo 仍遵守到返、餐窗与固定安排，无法核验必去地点时明确失败。
- 车行交通段使用 `PLANNING_COMPLETED` V5；V4 继续只接受步行，禁止就地扩展已发布版本。
