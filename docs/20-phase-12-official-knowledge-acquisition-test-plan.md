# Phase 12 官方知识采集测试计划

## 1. 目标与范围

本阶段把官方旅游资料获取拆成独立的采集边界。第一小步交付来源注册表、固定 URL 发现、域名白名单和基础 SSRF 防护；第二小步交付受控条件 HTTP 获取；第三小步交付 DNS 结果校验和单次抓取 IP 固定。当前仍不做开放式递归爬虫，不把候选内容直接发布到 RAG。

后续小步依次实现：限速与退避重试、采集快照、官方文章正文抽取、质量校验与审核队列、`KnowledgeImporter` 发布适配器和 freshness report。

## 2. 第一小步契约

- 来源配置使用仓库内 TOML，记录城市、官方域名、固定资源 URL、可靠性等级、抓取间隔和响应大小上限。
- URL 只允许 `https`、白名单主机和公网 IP；禁止用户名密码、非标准端口、内网/回环/保留地址。
- 来源配置必须有唯一 `source_id`，整个注册表中的资源 URL 规范化后不能重复。
- `SourceCatalog` 只负责加载和校验；发现器已经保留独立适配器边界，抓取器和快照存储将在后续小步通过 Protocol 解耦。
- CLI 校验失败时返回非零状态并输出机器可读 JSON，不进行网络请求。

## 3. 第二小步契约

- `FetchValidators` 显式承载 ETag 和 Last-Modified；校验器存在时发送 `If-None-Match` 和 `If-Modified-Since`。
- 304 返回 `ResourceNotModified`，不读取或伪造正文；2xx 返回带抓取时间、最终 URL、内容类型和校验器的 `ResourceFetched`。
- 请求使用 `Accept-Encoding: identity`，非 identity 响应在读取前拒绝；正文按原始字节流累计并执行 `max_response_bytes`，同时提前拒绝已声明超限的 `Content-Length`。
- HTTPX 禁止自动重定向；每一跳使用来源白名单重新校验，白名单外跳转在第二次请求前阻断。
- 超时、连接失败、429/5xx 和普通 4xx 统一映射为 `AcquisitionFetchError`，供后续调度器判断是否重试；本小步尚不执行重试。

## 4. 第四小步契约

- `KnowledgeSource` 显式配置每来源最小请求间隔，所有首次请求和重试都必须先取得该来源的时间槽；不同来源互不阻塞。
- `RetryPolicy` 限定最大尝试次数、初始退避和退避上限；只重试 `retryable=true` 的 `AcquisitionFetchError`，退避按上限封顶的指数序列执行。
- 调度器返回强类型成功或失败结果，并保留每次尝试的开始/结束时间、错误码、HTTP 状态和可重试标记；达到上限或遇到不可重试错误时，最终失败记录必须可供后续持久化。
- 单调时钟、UTC 时钟和异步休眠通过端口注入，自动化测试不得真实等待；本小步不创建采集数据库表。

## 5. 第五小步契约

- `knowledge_resource` 以来源 ID 和注册表中的规范化资源 URL 唯一标识受管资源，分别记录当前内容哈希、最近尝试、最近成功核验和最近内容变化时间；并发运行乱序完成时，不得用旧结果回退最新内容、校验器或核验时间。
- 每次调度执行都必须写入一条 `knowledge_fetch_run`，保留起止时间、全部尝试、最终状态和可重试错误元数据；资源、候选快照和运行记录在一个 PostgreSQL 事务中提交。
- `FETCHED` 对原始响应字节计算 SHA-256，并按资源、内容哈希和解析器版本幂等创建不可变 `PENDING` 候选快照；候选保留来源 URL、最终 URL、发布时间、抓取时间、内容类型、HTTP 校验器和解析器版本。
- `NOT_MODIFIED` 只更新资源的最近核验时间和校验器并记录运行，不创建候选快照；`FAILED` 只更新最近尝试时间并记录最终错误，不更新最近核验时间或创建快照。
- `AcquisitionWorkflow` 是生产执行入口，必须先读取已保存的条件校验器及其对应内容哈希，再组合调度与记录；304 必须携带这份基线哈希，避免并发边缘响应把校验器与当前内容拼错。本小步不调用 `KnowledgeImporter`，候选快照必须经过后续正文抽取、质量检查和人工审核，才能进入 RAG 发布边界。

## 6. 第六小步契约

- 广州政府页面使用 Beautiful Soup 解析 HTML，不用正则或字符串切片解析标签；HTTP `Content-Type` 声明的字符集优先于页面 meta，未知字符集拒绝。正文只从已知的 `#zoomcon`、`.content_article` 及标准 `main/article` 候选容器选择，过滤脚本、样式、分享控件、导航页脚和“相关附件/相关内容”后的尾部噪声。
- 标题、发布时间和内容来源优先读取 `ArticleTitle`、`PubDate`、`ContentSource` 元数据；无时区的官方时间按 `Asia/Shanghai` 解释后转 UTC。发布时间缺失只产生警告并保留为空，未来发布时间、标题缺失、正文容器缺失或正文过短必须拒绝。
- 正文以稳定的段落顺序输出纯文本，连续重复段落去重；包含价格、开放/营业时间、预约或交通等动态事实时保留内容但增加 `DYNAMIC_FACTS_PRESENT` 警告，后续发布和回答不得把它当作实时承诺。
- 解析器返回强类型 `EXTRACTED` 或 `REJECTED` 结果、固定解析器版本和结构化质量问题；本小步仍不修改原始快照，也不调用 `KnowledgeImporter`。

## 7. 第七小步契约

- `knowledge_extraction` 按 `(snapshot_id, parser_version)` 保存不可变解析结果；通过结果保留标题、正文、正文哈希、发布时间、内容来源与质量警告，拒绝结果只保留结构化错误，两类结果都保留稳定结果指纹和解析时间。
- 待处理查询只返回仍为 `PENDING` 且尚无指定解析器版本结果的原始快照；解析器升级可以为同一快照创建新结果，重复执行相同版本必须幂等，指纹变化必须报不可变冲突。
- `SnapshotExtractionService` 有界读取候选，在工作线程执行 HTML 解析并逐条持久化 `EXTRACTED`/`REJECTED`，单页质量拒绝不阻断同批其他页面；本小步不改变快照审核状态，也不调用 `KnowledgeImporter`。

## 8. 第八小步契约

- 只有无 `ERROR` 的 `EXTRACTED` 结果可以被人工批准或拒绝；审核动作追加保存操作者、UTC 时间、非空备注和稳定指纹，快照只允许从 `PENDING` 转为 `APPROVED` 或 `REJECTED`。批准还必须冻结分类、有效期、适用季节和旅客类型，并按资源串行分配稳定 `document_id` 与单调递增版本。
- 批准事务同时创建待发布记录；拒绝不创建发布记录。批准后、发布抢占前可以追加 `WITHDRAW` 动作并原子取消发布；已抢占或已发布的版本不能声明为成功撤回。
- `KnowledgeReviewPublisher` 是采集边界内唯一允许调用 `KnowledgeImporter.import_document` 的适配器。发布先按 PostgreSQL 时钟持久化抢占，再导入冻结的 `KnowledgeDocument`，最后回写结果；失败可重试，崩溃遗留的超时抢占可被重领，依靠 `KnowledgeImporter` 的文档版本事务锁与不可变幂等性收敛。任何发布尝试开始后都不得撤回，因为外部写入是否发生已不能由本地失败状态证明。
- 动态事实警告随提取结果保留；本小步只发布审核确认的静态知识，不改变“价格、营业时间、预约和交通必须实时核验”的回答约束。

## 9. 自动化测试

- 有效广州官方来源配置可加载，并能按城市筛选。
- 重复来源 ID、重复资源 URL、非 HTTPS URL 和不在白名单的主机被拒绝。
- 私有、回环、链路本地和保留 IP 被拒绝；来源配置不能携带 URL 凭据。
- 抓取策略边界（超时、间隔、最大响应体）在领域模型边界校验。
- CLI `validate` 输出来源和资源数量，错误配置返回失败。

第四小步结果：新增 21 项来源限速、重试和调度集成测试，覆盖指数退避封顶、尝试上限、不可重试终止、条件校验器传递、UTC 尝试记录、同来源间隔、跨来源独立、晚唤醒重检和取消恢复；NaN/Infinity 退避参数被拒绝。采集相关共 70 项测试；本机 Docker 未运行时 Python 全量为 205 项通过、4 项 pgvector 集成测试跳过，知识检索与采集总覆盖率 85.50%，通过 80% 门禁。

第五小步结果：新增 25 项记录/工作流与 PostgreSQL 集成测试，覆盖候选元数据、完整尝试审计、受管 URL 复核、版本化条件状态、304/失败分支、相同内容幂等、解析器升级、A→B→A 回退、实际并发与乱序完成保护、fetched-B/304-A 交错、事务回滚和并发迁移。采集相关共 95 项测试；真实 pgvector PostgreSQL 下 Python 全量 234 项通过，知识检索与采集总覆盖率 93.29%，通过 80% 门禁。

第六小步结果：新增 19 项 HTML 正文与质量规则测试，覆盖官方元数据、两类真实正文容器、HTTP/header-only/冲突 meta 的 UTF-8/GB18030、未知编码、日期/时间/显式时区、块级与通用容器噪声过滤、附件标记自然语句、段落去重、缺失发布时间告警、6 类动态事实表述和拒绝分支。3 个注册表真实页面均返回 200 并解析为 1421、850、7409 字符正文，标题和发布时间正确，沙面与 Citywalk 页标记 `DYNAMIC_FACTS_PRESENT`。采集相关共 114 项测试；真实 pgvector PostgreSQL 下 Python 全量 253 项通过，知识检索与采集总覆盖率 92.60%，通过 80% 门禁。同一资源首次并发插入的双唯一索引死锁已通过事务级资源锁修复，实际并发用例连续 20 轮通过。

第七小步结果：新增 10 项批处理与真实 PostgreSQL 测试，覆盖同批通过/拒绝结果、状态与质量严重级别不变量、稳定提取 ID/正文哈希/指纹、naive 发布时间拒绝、等价时区 UTC 指纹、批量上限、V2 迁移、待处理查询、重复 `unchanged`、合法指纹冲突和解析器升级重新处理。采集相关共 124 项测试；真实 pgvector PostgreSQL 下 Python 全量 263 项通过，知识检索与采集总覆盖率 92.81%，通过 80% 门禁。

当前安全边界已经覆盖 URL 字面 IP、协议、凭据、域名白名单、DNS 全结果公网单播校验、单次抓取 IP 固定和环境代理隔离，并在每次重定向前重复 URL 策略；新重定向主机还会重新执行 DNS 检查。

## 11. 后续验收边界

- freshness report 必须区分最近尝试、最近验证和最近内容变化，并按来源暴露过期原因。
- 价格、营业时间、预约和交通等动态事实不从静态采集内容直接承诺，回答时必须实时核验。
