# Phase 10 前端地图与时间轴联动测试计划

## 1. 目标与范围

本阶段把当前行程 API 已返回的活动坐标、地址和步行 polyline 接入 Vue 工作台，完成地图 Marker、路线和时间轴双向联动。

本阶段不实现路线编辑、公共交通、拖拽重排、POI 详情弹窗或地图搜索。真实高德 JS API 凭据由用户在本地单独配置；服务端 `AMAP_WEB_SERVICE_KEY` 不得进入浏览器构建或日志。

## 2. 行为契约

- 有合法坐标的活动按日期和活动顺序生成 Marker；无坐标活动保留在时间轴，但不进入地图模型。
- 每个交通段使用 API polyline 绘制路线，不在浏览器重新请求路线。
- 点击时间轴活动后，地图对应 Marker 高亮；真实地图模式同时定位并放大该地点。
- 点击地图 Marker 或地点列表后，时间轴对应活动滚动到可视区域并高亮。
- 缺少浏览器专用高德凭据、SDK 加载失败或活动无坐标时，页面必须显示稳定且可操作的降级状态。
- 桌面双栏、移动单栏均不得产生横向溢出、文字截断或控件重叠。

## 3. 凭据边界

前端只读取 Vite 公开变量：

```text
VITE_AMAP_WEB_JS_KEY
VITE_AMAP_SECURITY_CODE
```

变量位于仓库根目录、被 Git 忽略的 `.env`。`AMAP_WEB_SERVICE_KEY` 仅供 Python Provider 服务端调用，禁止复用为 JS API Key。

## 4. 自动化测试

- 地图纯函数测试：活动/路线扁平化、边界计算、坐标投影、无坐标与旧契约兼容。
- 组件测试：无凭据路线概览、可访问 Marker、选中事件、假的高德 SDK Marker/polyline 渲染、地图销毁。
- 应用测试：SSE 完成后同时展示地址、地图和时间轴；地图选择与时间轴选择双向同步。
- 完整 Vue 回归、TypeScript 类型检查和生产构建。

## 5. 浏览器验收

- 1280 x 800 桌面视口：地图与时间轴双栏，无横向溢出，初始选中状态一致。
- 390 x 844 移动视口：地图与时间轴纵向排列，无重叠、越界或横向滚动。
- 依次点击地图 Marker 和时间轴活动，验证双方 `aria-pressed` 状态同步。
- 检查控制台 warning/error，并检查关键文字和按钮边界。

## 6. 实施结果

- 新增地图模型、坐标投影和高德 JS API 懒加载适配器。
- 新增 `TripMap`，支持真实高德 Marker/polyline 与无需凭据的路线概览。
- 时间轴活动改为可访问的选择按钮，显示地址，并与地图共用活动 ID 选中状态。
- Vue 43 个测试全部通过；地图切片语句/行覆盖率 92.66%、分支覆盖率 81.63%、函数覆盖率 83.33%，覆盖阈值固定为 80%。
- TypeScript 类型检查和生产构建通过。
- 桌面与移动浏览器验收通过：无横向溢出、元素越界、重叠或控制台错误。
